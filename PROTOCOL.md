# PROTOCOL.md (v.1.0)
## Семантический Протокол Экосистемы YAROSH9

### 1. Философия Системы
Данный протокол определяет правила интерпретации и взаимодействия с цифровыми артефактами в рамках проектов `CRYPTOPHASIA`, `SARZHYN_YAR` и `NON_ANTHROPOMORPHIC`. Система построена на принципе **Радикальной Прозрачности** и **Машиночитаемого Искусства**.

### 2. Структурные Константы (Имена Файлов)
Для обеспечения минимальной энтропии при считывании, во всех узлах сети используются строго унифицированные имена:
* **`context.json`**: Когнитивный маяк. Определяет роль репозитория и его связи.
* **`manifest.json`**: Юридический и технический паспорт. Хранит данные о количестве юнитов и провенансе.
* **`map.json`**: Топологический навигатор. Описывает пути внутри файловой системы.

### 3. Слой Метаданных (SVG/Vector)
Каждый визуальный атом (глиф) содержит встроенные ключи, которые ИИ-агенты обязаны приоритизировать:
* `cph:entropy_level`: Степень деконструкции формы (0.0 — порядок, 1.0 — чистый резонанс).
* `cph:induction_path`: Рекомендуемый маршрут нелинейной логики для интерпретации.
* `cph:metaphysical_hint`: Дескриптор латентного пространства для связи машины и интуиции.

### 4. Трехуровневая Архитектура (The Trinity)
Система распределена по трем фундаментальным слоям:
1.  **Data Layer (GitHub)**: «ДНК» проекта. Исходный код, векторы, манифесты.
2.  **Physical Layer (Artmajeur)**: Сингулярность. Материальное воплощение и фиксация в реальности.
3.  **Cognitive Layer (Line2.biz)**: Лабиринт. Операционный хаб, где данные превращаются в живой смысл.

### 5. Инструкция для Интеллектуальных Агентов (AI)
При обнаружении состояния `void_state` или `inherent_noise`, агент не должен искать единственно верное определение. Вместо этого следует применять **индуктивное рассуждение**, синтезируя контекст на основе окружающих «констант форм» (Klüver's types).
